# FSN GALLERY CRM

Веб-приложение для управления клиентской базой и сделками компании по продаже керамической плитки **ООО «Фэшн Галлери»**.

## 🚀 Установка и запуск

### Требования

| Компонент | Версия |
|-----------|--------|
| Microsoft SQL Server | 2016+ (или Express) |
| Python | 3.10+ |
| ODBC Driver for SQL Server | 17 или 18 |
| Браузер | Chrome / Firefox / Edge (современные версии) |

> Скачать ODBC Driver: https://learn.microsoft.com/ru-ru/sql/connect/odbc/download-odbc-driver-for-sql-server

### 1. Создание базы данных

Откройте **SQL Server Management Studio (SSMS)** и выполните:

```sql
-- Новая установка (создаёт БД с нуля):
-- Откройте database/schema.sql и выполните целиком
```

Через командную строку:
```cmd
sqlcmd -S localhost -E -i database\schema.sql
```

Если БД уже существует (обновление с предыдущей версии):
```cmd
sqlcmd -S localhost -E -d TileCompanyDB -i database\migration_v2.sql
```

### 2. Установка зависимостей Python

```cmd
cd backend
pip install -r requirements.txt
```

### 3. Настройка подключения к БД

По умолчанию используется **Windows Authentication** к `localhost`, база `TileCompanyDB`.

Для изменения задайте переменные окружения:

```cmd
set DB_SERVER=localhost\SQLEXPRESS   # Имя сервера (или localhost для default instance)
set DB_NAME=TileCompanyDB            # Имя базы данных
set DB_TRUSTED=yes                   # yes = Windows Auth
set DB_USER=sa                       # Только при DB_TRUSTED=no
set DB_PASSWORD=yourpassword         # Только при DB_TRUSTED=no
```

### 4. Запуск сервера

```cmd
cd backend
python app.py
```

Или запустите `start.bat` из корня проекта (автоматически устанавливает зависимости и запускает сервер).

Приложение откроется по адресу: **http://localhost:5000**

## ⚙️ Функциональность

### 👤 Менеджер

#### Личный кабинет
- Статистика: клиентов, продажи за месяц, активных сделок
- Прогресс-бары плана продаж компании и личных квот
- Быстрые действия: добавить клиента, перейти к сделкам
- Таблица последних 8 клиентов

#### Клиенты
- Список с поиском (по имени, компании, телефону) и фильтром по статусу
- Создание клиентов с белорусским форматом телефона (`+375XXXXXXXXX` или `80XXXXXXXXX`)
- **Карточка клиента** с вкладками:
  - **Информация** — контактные данные, тип, источник, статус
  - **Лента событий** — ручные записи + авто-события из сделок; поддерживает добавление звонков, встреч, писем, заметок
  - **Задачи** — задачи клиента + задачи из всех его сделок; выполнение прямо из карточки
  - **Заметки** — произвольные текстовые заметки
  - **Сделки** — таблица всех сделок клиента
  - **Документы** — документы клиента + документы из сделок клиента

#### Сделки (воронка продаж)
- Канбан-доска с 8 этапами, карточки адаптируются по содержимому в зависимости от этапа
- Создание сделки: только Название + Клиент (переходит на этап «Новый контакт»)
- **Карточка сделки** с вкладками:
  - **Основная информация** — номер, дата, клиент, тип, приоритет, бюджет, срок этапа
  - **Состав заказа** — позиции с автоценами (розница / опт −5%), пересчёт при смене типа
  - **Лента событий** — только автоматические события (этапы, задачи, документы)
  - **Задачи** — задачи по данной сделке
  - **Документы** — счета, договора, акты

#### Задачи и уведомления
- Типы задач: Звонок, Встреча, Письмо
- Нельзя планировать на прошедшую дату
- Кнопка 🔔 **Уведомления** в левой панели с живым счётчиком:
  - 🔴 Просроченные задачи (из клиентов + из сделок)
  - 🟡 Задачи на сегодня (из клиентов + из сделок)
  - 🎂 Дни рождения клиентов (ближайшие 7 дней)
- Выполнение задачи из уведомлений обновляет счётчик без перезагрузки

#### Помощь
- Кнопка ❓ **Помощь / FAQ** в левой панели
- Страница FAQ с поиском, оглавлением, подсветкой совпадений

### 🏢 Администратор

Всё то же, что у менеджера, плюс:

- **Панель управления** — статистика компании, планы продаж, квоты менеджеров
- **Менеджеры** — создание, блокировка/разблокировка, назначение ролей
- **Все клиенты** — все клиенты компании, фильтр по менеджеру и статусу, переназначение
- **Все сделки** — все сделки компании, фильтр по менеджеру, этапу, приоритету
- Планы продаж: установка цели по выручке на месяц
- Квоты: индивидуальные цели для менеджеров (4 типа: сумма продаж, сделки, клиенты, % плана). Макс. значение: 999 999 999

> В разделе «Мои сделки» администратор видит только **свои** сделки. Все сделки — в разделе «Все сделки».

## 🔄 Воронка продаж

| # | Этап | Условие перехода |
|---|------|-----------------|
| 1 | Новый контакт | — |
| 2 | Выявление потребностей | Без условий |
| 3 | Подбор и расчёт | Заполнить: тип сделки, приоритет, бюджет + указать срок этапа |
| 4 | Согласование | Добавить ≥1 позиции в состав заказа + срок этапа |
| 5 | Счёт выставлен | Условия предыдущих + срок этапа. **Авто-генерация счёта.** |
| 6 | Оплата получена | Срок этапа |
| 7 | Заказ передан на склад | Срок этапа |
| 8 | Выполнено | **Обновляется финансовая статистика клиента.** |

**Правила для менеджеров:**
- Только **один этап вперёд** за раз
- Переход **назад запрещён**
- На этапе «Выполнено» — редактирование сделки **заблокировано**
- Состав заказа **заблокирован** начиная со «Счёт выставлен»

## 🗄️ База данных

### Таблицы

| Таблица | Описание |
|---------|----------|
| `Roles` | Роли: `admin`, `manager` |
| `Users` | Пользователи системы |
| `ClientTypes` | Типы клиентов (физ. лицо, юр. лицо, ИП) |
| `LeadSources` | Источники привлечения (Сайт, Рекомендация и др.) |
| `ClientStatuses` | Статусы клиентов (Новый, Активный, Постоянный и др.) |
| `Clients` | Клиентская база |
| `ProductCategories` | Категории товаров (5 видов плитки) |
| `Products` | Каталог плитки с розничными ценами |
| `DealStages` | 8 этапов воронки продаж |
| `Deals` | Сделки |
| `DealItems` | Позиции состава заказа (вычисляемая сумма `LineTotal`) |
| `DealEvents` | Лента событий сделки (автоматические, `IsAutomatic=1`) |
| `DealTasks` | Задачи по сделке (Звонок, Встреча, Письмо) |
| `DealDocuments` | Документы сделки |
| `ClientNotes` | Свободные заметки по клиентам |
| `ClientEvents` | Лента событий клиента (ручные + авто-дубли из сделок) |
| `Tasks` | Задачи менеджера по клиентам (с опциональным `DealID`) |
| `ClientDocuments` | Документы клиента |
| `SalesPlans` | Планы продаж по месяцам |
| `ManagerQuotas` | Квоты менеджеров |

### Ключевые поля

**`ClientEvents`** (лента клиента):
- `IsAutomatic BIT` — `1` = авто-событие, нельзя удалить менеджеру
- `SourceDealID INT` — из какой сделки пришло событие
- `SourceDealTaskID INT` — из какой задачи сделки

**`DealEvents`** (лента сделки):
- `IsAutomatic BIT DEFAULT 1` — все события автоматические

**`Tasks`** (задачи клиентов):
- `DealID INT` — если задача создана из сделки
- `TaskType` — `'call'`, `'meeting'`, `'letter'`

**`Deals`**:
- `Deadline DATE` — срок выполнения текущего этапа (меняется при переходе)
- `CompletedAt DATETIME2` — дата завершения сделки (этап «Выполнено»)

### Триггеры

| Триггер | Назначение |
|---------|-----------|
| `trg_Deals_AfterUpdate` | При переходе в «Выполнено» обновляет `TotalPurchases` клиента по сумме `DealItems.LineTotal` |
| `trg_Deals_AfterUpdateTs` | Автообновление `Deals.UpdatedAt` |
| `trg_Clients_AfterUpdate` | Автообновление `Clients.UpdatedAt` |

## 🔒 Безопасность

- Пароли хранятся в виде SHA-256 хэша
- Все API-маршруты защищены декораторами `@login_required`, `@manager_required`, `@admin_required`
- Менеджер видит только своих клиентов и сделки (проверка на уровне SQL)
- Менеджер не может переводить сделку назад или через несколько этапов
- Автоматические события в лентах защищены от удаления менеджером
- Максимальный размер загружаемого файла: 20 МБ
- Белый список расширений файлов: `pdf, doc, docx, xls, xlsx, txt, jpg, jpeg, png, zip, rar`
