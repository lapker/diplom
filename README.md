## 2. Пошаговая инструкция установки

### 2.1 Шаг 1: Установка Microsoft SQL Server

1. Скачайте **SQL Server Express** с официального сайта Microsoft (бесплатная версия).
2. При установке выберите:
   - Тип установки: **Basic** (базовая) или **Custom** (выборочная)
   - Экземпляр: **Default instance** или **Named instance** (запомните имя)
3. Настройте аутентификацию:
   - Для **Windows Authentication** — используется учётная запись Windows
   - Для **Mixed Mode** — запомните пароль системного администратора (sa)
4. Завершите установку.

### 2.2 Шаг 2: Установка SQL Server Management Studio (SSMS)

1. Скачайте SSMS с официального сайта Microsoft.
2. Установите с параметрами по умолчанию.
3. Запустите SSMS и подключитесь к серверу.

### 2.3 Шаг 3: Установка ODBC Driver

1. Скачайте **ODBC Driver 17 for SQL Server** или **ODBC Driver 18**.
2. Установите с параметрами по умолчанию.
3. Перезагрузите компьютер после установки.

### 2.4 Шаг 4: Установка Python

1. Скачайте Python 3.10+ с официального сайта python.org.
2. При установке **обязательно** отметьте галочку **«Add Python to PATH»**.
3. Выберите **«Install Now»** (полная установка).
4. Проверьте установку: откройте командную строку и введите `python --version`.

### 2.5 Шаг 5: Создание базы данных

#### Вариант A: Новая установка (создание БД с нуля)

Через SQL Server Management Studio:
1. Откройте файл `database/schema.sql` в SSMS.
2. Выполните скрипт целиком (клавиша F5 или кнопка «Execute»).
3. После выполнения появится сообщение об успешном создании базы данных.

Через командную строку:
```cmd
sqlcmd -S localhost -E -i database\schema.sql
```

#### Вариант B: Обновление существующей базы данных

```cmd
sqlcmd -S localhost -E -d TileCompanyDB -i database\migration.sql
```

### 2.6 Шаг 6: Установка зависимостей Python

Откройте командную строку в корневой папке проекта и выполните:

```cmd
cd backend
pip install -r requirements.txt
```

### 2.7 Шаг 7: Настройка подключения к базе данных

По умолчанию приложение использует Windows Authentication к `localhost`, база данных `TileCompanyDB`.

Для изменения настроек задайте переменные окружения:

```cmd
# Для Windows Authentication (по умолчанию):
set DB_SERVER=localhost
set DB_NAME=TileCompanyDB
set DB_TRUSTED=yes

# Для SQL Server Authentication (с паролем):
set DB_SERVER=localhost
set DB_NAME=TileCompanyDB
set DB_TRUSTED=no
set DB_USER=sa
set DB_PASSWORD=yourpassword

# Для именованного экземпляра:
set DB_SERVER=localhost\SQLEXPRESS
```

Или отредактируйте файл `backend/app.py`, изменив значения по умолчанию в словаре `DB_CONFIG`.

### 2.8 Шаг 8: Запуск приложения

#### Способ 1: Использование скрипта запуска (рекомендуется)

Просто дважды кликните на файл `start.bat` в корневой папке проекта.

#### Способ 2: Ручной запуск

```cmd
cd backend
python app.py
```

### 2.9 Шаг 9: Доступ к приложению

После запуска сервера откройте веб-браузер и перейдите по адресу:

**http://localhost:5000**

- **Клиентская часть (Frontend):** Статические HTML-страницы с JavaScript (ES2020). Клиент отправляет API-запросы к серверу и отрисовывает пользовательский интерфейс.

- **Файловый сервер:** Локальная файловая система для хранения загруженных документов.

## 3. Описание основных функций

### 3.1 Функциональность менеджера

#### 3.1.1 Авторизация и личный кабинет

**Запуск и авторизация:**
1. Откройте приложение в браузере по адресу http://localhost:5000
2. Введите логин и пароль в соответствующие поля
3. Нажмите кнопку «Войти»
4. При успешной авторизации откроется личный кабинет (панель управления)

**Выход из системы:**
1. Нажмите на имя пользователя в левой панели
2. Выберите пункт «Выход» или нажмите кнопку выхода

**Личный кабинет содержит:**
- Статистику: количество клиентов, продажи за месяц, активные сделки
- Прогресс-бары выполнения плана продаж компании и личных квот
- Быстрые действия: добавить клиента, перейти к сделкам
- Таблицу последних 8 клиентов

#### 3.1.2 Управление клиентами

**Список клиентов:**
1. Перейдите в раздел «Клиенты» через левую панель
2. Используйте поле поиска для фильтрации по имени, компании или телефону
3. Выберите фильтр по статусу клиента из выпадающего списка
4. Нажмите на строку клиента для просмотра карточки

**Создание клиента:**
1. Нажмите кнопку «+ Новый клиент» или «Добавить клиента»
2. Заполните обязательные поля:
   - ФИО (полное имя)
   - Телефон (формат: +375XXXXXXXXX или 80XXXXXXXXX)
   - Тип клиента (физическое/юридическое лицо, ИП)
   - Источник привлечения
3. Заполните необязательные поля по необходимости
4. Нажмите кнопку «Сохранить»

**Карточка клиента содержит вкладки:**
- **Информация** — контактные данные, тип, источник, статус
- **Лента событий** — ручные записи и автоматические события из сделок. Для добавления события нажмите «+ Добавить», выберите тип (звонок, встреча, письмо, заметка), укажите заголовок и описание.
- **Задачи** — задачи клиента и задачи из всех его сделок. Для создания задачи нажмите «+ Новая задача», выберите тип (звонок, встреча, письмо), укажите заголовок и дату/время.
- **Заметки** — произвольные текстовые заметки
- **Сделки** — таблица всех сделок клиента
- **Документы** — документы клиента и документы из его сделок

**Редактирование клиента:**
1. Откройте карточку клиента
2. Нажмите кнопку «Редактировать»
3. Измените необходимые поля
4. Нажмите «Сохранить»

#### 3.1.3 Управление сделками (воронка продаж)

**Канбан-доска сделок:**
1. Перейдите в раздел «Мои сделки»
2. Воронка отображает 8 этапов: Новый контакт → Выявление потребностей → Подбор и расчёт → Согласование → Счёт выставлен → Оплата получена → Заказ передан на склад → Выполнено
3. Перетаскивайте карточки между этапами для перехода сделки вперёд
4. Нажмите на карточку для просмотра/редактирования сделки

**Создание сделки:**
1. Нажмите кнопку «+ Новая сделка» на странице «Мои сделки»
2. Укажите название сделки и выберите клиента
3. Нажмите «Создать» — сделка появится на этапе «Новый контакт»

**Карточка сделки содержит вкладки:**
- **Основная информация** — номер, дата, клиент, тип (розница/опт), приоритет, бюджет, срок этапа
- **Состав заказа** — позиции с автоценами. Для розницы — розничная цена, для опта — розничная цена минус 5%. Пересчёт происходит автоматически при смене типа сделки.
- **Лента событий** — автоматические события (этапы, задачи, документы)
- **Задачи** — задачи по данной сделке
- **Документы** — счета, договоры, акты

**Правила перехода по этапам:**
- Только один этап вперёд за раз
- Переход назад запрещён
- На этапе «Выполнено» редактирование заблокировано
- Состав заказа заблокирован начиная со «Счёт выставлен»

#### 3.1.4 Управление задачами и уведомлениями

**Типы задач:**
- Звонок (call)
- Встреча (meeting)
- Письмо (letter)

**Создание задачи:**
1. Откройте карточку клиента или сделки
2. Перейдите на вкладку «Задачи»
3. Нажмите «+ Новая задача»
4. Выберите тип задачи
5. Укажите заголовок и запланированную дату/время
6. Нажмите «Создать»

> **Важно:** Нельзя планировать задачу на прошедшую дату.

**Выполнение задачи:**
1. Перейдите в раздел «Уведомления» через кнопку 🔔 в левой панели
2. Найдите задачу в списке
3. Нажмите кнопку «Выполнить» рядом с задачей
4. Задача будет отмечена как выполненная, счётчик обновится автоматически

**Центр уведомлений отображает:**
- 🔴 Просроченные задачи (из клиентов и из сделок)
- 🟡 Задачи на сегодня (из клиентов и из сделок)
- 🎂 Дни рождения клиентов (ближайшие 7 дней)

#### 3.1.5 Помощь и FAQ

1. Нажмите кнопку ❓ «Помощь / FAQ» в левой панели
2. Используйте поле поиска для нахождения нужного раздела
3. Нажмите на заголовок вопроса для разворачивания ответа

### 3.2 Функциональность администратора

Администратор имеет доступ ко всем функциям менеджера, а также к дополнительным возможностям:

#### 3.2.1 Панель управления

- Статистика всей компании
- Управление планами продаж
- Назначение квот менеджерам

#### 3.2.2 Управление пользователями (Менеджеры)

**Создание менеджера:**
1. Перейдите в раздел «Менеджеры»
2. Нажмите «+ Добавить менеджера»
3. Заполните данные: логин, пароль, ФИО, email, телефон
4. Выберите роль: менеджер или администратор
5. Нажмите «Создать»

**Блокировка/разблокировка менеджера:**
1. В списке менеджеров найдите нужного пользователя
2. Нажмите кнопку «Заблокировать» или «Разблокировать»

> **Примечание:** Заблокированный менеджер не сможет войти в систему.

**Активация зарегистрированного пользователя:**
1. При самостоятельной регистрации новый пользователь ожидает назначения роли
2. Администратор должен назначить роль в разделе «Менеджеры»
3. После назначения роли и активации пользователь сможет войти

#### 3.2.3 Все клиенты компании

- Просмотр всех клиентов всех менеджеров
- Фильтрация по менеджеру и статусу
- Переназначение клиента другому менеджеру

#### 3.2.4 Все сделки компании

- Просмотр всех сделок всех менеджеров
- Фильтрация по менеджеру, этапу, приоритету

#### 3.2.5 Управление планами продаж

1. Перейдите в раздел «Планы продаж»
2. Выберите год и месяц
3. Укажите целевую сумму продаж
4. Нажмите «Сохранить»

#### 3.2.6 Управление квотами менеджеров

**Типы квот:**
- `sales_amount` — сумма продаж (BYN)
- `deals_count` — количество сделок
- `new_clients` — количество новых клиентов
- `plan_percent` — процент выполнения плана (%)

**Установка квоты:**
1. Перейдите в раздел «Квоты»
2. Выберите менеджера, год и месяц
3. Добавьте квоты нужных типов
4. Укажите целевое значение (максимум: 999 999 999)
5. Нажмите «Сохранить»

### 3.3 Временные характеристики

| Операция | Примерное время выполнения |
|----------|---------------------------|
| Авторизация | 0.5–2 сек |
| Загрузка списка клиентов (до 100) | 1–3 сек |
| Создание/обновление клиента | 0.5–1 сек |
| Переход сделки по этапу | 0.5–2 сек |
| Загрузка страницы карточки | 1–3 сек |
| Поиск по клиентам | 0.5–2 сек |
| Загрузка файла (до 20 МБ) | 2–10 сек |

### 3.4 Режимы работы

- **Однопользовательский режим:** Каждый пользователь работает со своей клиентской базой
- **Многопользовательский режим:** Несколько менеджеров работают одновременно, администратор распределяет доступ
- **Автономный режим:** Не поддерживается (требуется подключение к серверу)

3. Проверьте целостность базы данных запросом:
